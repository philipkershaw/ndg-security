connect(self, createServerSess=True, username=None, passphrase=None, userCert=None, sessID=None)

source code  
Create a new user session or connect to an existing one:

connect([createServerSess=True/False, ]|[, username=u, passphrase=p]|
        [, userCert=px]|[, sessID=id])

@type createUserSess: bool
@param createServerSess: If set to True, the SessionMgr will create
and manage a session for the user.  For command line case, it's 
possible to choose to have a client or server side session using this 
keyword.

@type username: string
@param username: username of account to connect to

@type passphrase: string
@param passphrase: pass-phrase - user with username arg

@type userCert: string
@param userCert: connect to existing session with proxy certificate
corresponding to user.  username/pass-phrase not required

@type sessID: string
@param sessID: connect to existing session corresponding to this ID.
username/pass-phrase not required.

@rtype: tuple
@return user certificate, private key, issuing certificate and 
session ID respectively.  Session ID will be none if createUserSess 
keyword is set to False

getAttCert(self, userCert=None, sessID=None, encrSessMgrURI=None, **credWalletKw)

source code 
For a given user, request Attribute Certificate from an Attribute Authority given by service URI. If sucessful, an attribute certificate is added to the user session credential wallet and also returned from this method
Parameters:
  • userCert (string) - user's certificate to key into their session
  • sessID (string) - user's ID to key into their session
  • encrSessMgrURI (string) - URI for remote session manager to forward a request to. This effectively use THIS session manager as a proxy to another. This URI is encrypted with a shared key. The key is stored in the property file 'sessMgrEncrKey' element. *** This functionality is redundant for NDG BETA delivery ***
  • **credWalletKw - keywords to CredWallet.getAttCert
