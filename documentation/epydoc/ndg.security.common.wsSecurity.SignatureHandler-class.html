Parameters:
  • verifyingCert (M2Crypto.X509.X509 / ndg.security.common.X509.X509Cert) - X.509 certificate used by verify method to verify a message. This argument can be omitted if the message to be verified contains the X.509 certificate in the BinarySecurityToken element. In this case, the cert read from the message will be assigned to the verifyingCert attribute.
  • verifyingCertFilePath (string) - alternative input to the above, pass the file path to the certificate stored in a file
  • signingCert (M2Crypto.X509.X509 / ndg.security.common.X509.X509Cert) - certificate associated with private key used to sign a message. The sign method will add this to the BinarySecurityToken element of the WSSE header. binSecTokValType attribute must be set to 'X509' or 'X509v3' ValueTyep. As an alternative, use signingCertChain - see below...
  • signingCertFilePath (string) - alternative input to the above, pass the file path to the certificate stored in a file
  • signingCertChain (list or tuple) - pass a list of certificates constituting a chain of trust from the certificate used to verifying the signature backward to the CA cert. The CA cert need not be included. To use this option, reqBinSecTokValType must be set to the 'X509PKIPathv1' ValueType
  • signingPriKey (M2Crypto.RSA.) - private key used to be sign method to sign message
  • signingPriKeyFilePath (string) - equivalent to the above but pass private key from PEM file
  • signingPriKeyPwd (string or None) - password protecting private key. Set / default to None if there is no password.
  • caCertDirPath (string) - establish trust for signature verification. This is a directory containing CA certificates. These are used to verify the certificate used to verify the message signature.
  • caCertFilePathList (list or tuple) - same as above except pass in a list of file paths instead of a single directory name.
  • refC14nKw (dict) - dictionary of keywords to reference Canonicalization. Use 'unsuppressedPrefixes' keyword to set unsuppressedPrefixes.
  • signedInfoC14nKw (dict) - keywords to Signed Info Canonicalization. It uses the same format as refC14nKw above.
  • __beginCert (string) - delimiter for beginning of base64 encoded portion of a PEM encoded X.509 certificate

__getVerifyingCert(self)

source code  
Return X.509 cert object corresponding to cert used to verify the 
signature in the last call to verify

 * Cert will correspond to one used in the LATEST call to verify, on
 the next call it will be replaced
 * if verify hasn't been called, the cert will be None

@rtype: M2Crypto.X509.X509
