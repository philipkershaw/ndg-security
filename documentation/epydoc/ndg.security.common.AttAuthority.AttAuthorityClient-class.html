getAttCert(self, userId=None, userCert=None, userAttCert=None)

source code 
Request attribute certificate from NDG Attribute Authority Web Service.
Parameters:

  • userId (string) - DN of the X.509 certificate used in SOAP digital signature corresponds to the *holder* of the Attribute Certificate that is issued. Set this additional field to specify an alternate user ID to associate with the AC. This is useful in the case where, as in the DEWS project, the holder will be a server cert. rather than a user proxy cert.

    If this keword is omitted, userId in the AC will default to the same value as the holder DN.
  • userCert (string) - certificate corresponding to proxy private key and proxy cert used to sign the request. Enables server to establish chain of trust proxy -> user cert -> CA cert. If a standard private key is used to sign the request, this argument is not needed.

  • userAttCert (string / AttCert) - user attribute certificate from which to make a mapped certificate at the target attribute authority. userAttCert must have been issued from a trusted host to the target. This is not necessary if the user is registered at the target Attribute Authority.

    @rtype ndg.security.common.AttCert.AttCert @return attribute certificate for user. iIf access is refused, AttributeRequestDenied is raised
